1. Business Context

1.1 Problem Domain

Sistem mengelola full lifecycle campaign DCB (direct carrier billing):

  • Campaign tracking: traffic → landing → click → MO → billed
  • Postback handling dari adnet callback
  • Performance reporting: CPA, ARPU, traffic, revenue, conversion, alert
  • Budget management: target budget, IO approval, agency fee
  • Master data: country, operator, partner, service, adnet, agency, channel
  • User & access control: user, role, menu, permission, company hierarchy

3.2 Use Case Detail (sample)

UC-01: Receive Postback from Adnet

Actor: Adnet · Trigger: HTTP GET /v1/postback?...

Goal: Adnet melaporkan event billed/unbilled untuk record campaign tracking + trigger downstream pixel storage.

Main flow:
  1. Adnet hit endpoint dgn URL params (campaign_id, partner, msisdn, status)
  2. Sistem validasi params (no auth header — public)
  3. Sistem cek IP allowlist (kalau aktif)
  4. Insert ke postbacks table
  5. Publish ke RabbitMQ E_PIXELSTORAGE
  6. Return HTTP 200 + tracking response
Postcondition: Record postback tersimpan, message di queue siap dikonsumsi worker.

UC-05: Edit Campaign Capping/Ratio/Postback

Actor: AM, Admin · Trigger: POST /v1/management/campaign/editratio (atau editmocapping, editpo, editcampaign)

Goal: AM update parameter campaign berjalan tanpa restart.

Main flow:
  1. AM submit form dgn campaign_id + nilai baru
  2. JWT divalidasi → user_id + scope
  3. Cek user authorize untuk campaign (via company/agency scope)
  4. Update campaign_details table
  5. Publish ke E_CAMPAIGNMANAGEMENT (worker propagasi ke landing service)
  6. Return updated record

8. Acceptance Criteria

8.1 Per Use Case

UC-01 (Postback)

  • Adnet GET /v1/postback?campaign_id=X&... → response 200 dlm < 500 ms
  • Record postback insert ke postbacks dgn semua param
  • Message ter-publish ke E_PIXELSTORAGE
  • Duplicate postback dlm 600s tidak ter-record dua kali
  • Postback gagal validate → 400 + log warn dgn payload

UC-05 (Edit campaign)

  • AM scope agency X bisa edit campaign agency X
  • AM scope agency X tidak bisa edit campaign agency Y → 403
  • Update tersimpan ke campaign_details + audit log (kalau ada)
  • Message ter-publish ke E_CAMPAIGNMANAGEMENT
  • Concurrent edit oleh 2 AM → last-write-wins (sementara, TODO lock)

UC-15 (Logout)

  • Logout endpoint call h.RevokeJWT(jti, ttl) dgn ttl = exp - now
  • Token sama dipakai → 401 Token revoked
  • Setelah ttl expire, key blacklist auto-cleanup oleh Redis

8.2 Cross-Cutting (Definition of Done)

Untuk setiap fitur baru, checklist berikut wajib sebelum merge:
  • Endpoint terdaftar di src/app/routes/<domain>.go
  • Handler di src/handler/incoming_<domain>_handler.go
  • Repository di src/domain/repository/<domain>.go
  • Entity (kalau perlu) di src/domain/entity/<domain>.go + register di migrateEntities
  • JWT auth applied (kalau bukan public)
  • Scope filter (company/agency/adnet) sesuai actor
  • Error response konsisten dgn envelope {code, desc, error}
  • Response time < 1s read, < 3s export
  • Logs structured (logrus) — no println
  • Secrets tidak di-log (Cfg.Redacted() pattern)
  • staticcheck ./... zero new warning
  • go build ./... exit 0
  • SPEC.md + SPEC.html updated (use case, API, entity, AC)

8.3 Non-Functional

  • DB query log enabled — no N+1 baru
  • Endpoint baru di-monitor di Grafana
  • Postback endpoint stress test ≥ 500 req/sec
  • Redis fail simulation: degraded mode tidak crash
  • DB pool sat test: tidak melebihi DB_MAX_OPEN_CONNS

9. Add Feature — Quick Workflow

1. Identify domain (campaign/budget/master-data/...)

2. Update SPEC (md + html):
   - Section 3: Use case
   - Section 4.4: Endpoint
   - Section 5: Entity (kalau perlu)
   - Section 7: Edge case
   - Section 8.1: Acceptance criteria

3. Implement:
   - entity   → src/domain/entity/<file>.go (+ register migrateEntities)
   - repo     → src/domain/repository/<file>.go
   - handler  → src/handler/incoming_<file>_handler.go
   - route    → src/app/routes/<group>.go
   - DTO      → src/domain/entity/<file>.go (request/response struct)

4. Test:
   - go build ./...
   - staticcheck ./...
   - manual hit endpoint
   - cek log untuk leak secret

5. Run migration:
   ./datasource migrate

6. Deploy:
   - tag git v1.x.y
   - CI auto build
   - update compose version
   - run migrate compose first
   - deploy server
Pre-prod reminder: AES_SECRET_KEY rotate & pindah ke secret manager. AUTH_ENFORCE_DEFAULT=true setelah FE migrate. Multi-replica deploy: pastikan DB_MAX_OPEN_CONNS × replicas ≤ PG max_connections.
